# E-commerce Sales Analysis — Data Analysis Project

Week 4 Project: a complete data analysis pipeline that loads, cleans, analyzes,
and visualizes six months of e-commerce order data, and produces a written
insights report.

## 1. Project Overview

**Schema:** `Date, Product, Quantity, Price, Customer_ID, Region, Total_Sales`

**Goal:** Analyze order-level sales data to answer three questions:
1. Which products actually drive revenue? (revenue by product)
2. Which regions perform best? (revenue by region)
3. How is revenue trending month to month?

**What it does:**
- Loads raw order data from a CSV file
- Cleans it (missing values, invalid quantities/prices, mismatched totals, duplicates)
- Computes summary statistics (total revenue, product/region breakdown, monthly trend, MoM change, revenue per customer)
- Generates 3 charts: a bar chart, a pie chart, and a line chart
- Writes a Markdown report combining the numbers, the charts, and written insights

The dataset is synthetic (generated by `data/generate_data.py`) but modeled on
realistic e-commerce orders: 8 products (Phone, Laptop, Tablet, Headphones,
Smartwatch, Charger, Speaker, Camera) across 4 regions (East, West, North,
South), Jan–Jun 2024. It's deliberately seeded with messy data (missing
fields, negative quantities, a duplicate row, a `Total_Sales` value that
doesn't match `Quantity x Price`, inconsistent region capitalization) so the
cleaning step has real work to do — this mirrors what real sales exports
usually look like.

`main.py` is organized as a linear pipeline with one function per stage:

| Function | Responsibility |
|---|---|
| `load_data()` | Reads the CSV, validates it exists, isn't empty, and has all 7 required columns |
| `clean_data()` | Handles missing Product/Region, invalid Quantity/Price, mismatched `Total_Sales`, unknown regions, unparseable dates, and duplicates |
| `analyze_data()` | Computes total revenue, product/region breakdown, monthly trend, average order value, revenue per customer |
| `create_bar_chart()` / `create_pie_chart()` / `create_line_chart()` | Generate and save each chart as a PNG |
| `write_report()` | Assembles all of the above into `report/analysis_report.md` |
| `main()` | Orchestrates the pipeline with error handling at each stage |

## 5. Technical Details

**Data cleaning rules (`clean_data()`):**
- Region text is stripped and standardized to title case, so `"south"`,
  `" South "`, and `"SOUTH"` all become `"South"`.
- Product text is stripped; blank/`"nan"`/`"none"` values are treated as missing.
- Rows with missing Product or Region are dropped.
- `Quantity`, `Price`, and `Total_Sales` are coerced to numeric; anything
  that can't be converted is dropped.
- Zero or negative Quantity/Price is dropped (invalid transactions).
- Only known regions (East, West, North, South) are kept.
- **`Total_Sales` is cross-checked against `Quantity x Price`.** If they
  don't match (beyond a tiny rounding tolerance), `Total_Sales` is
  recalculated from `Quantity x Price` rather than dropping the row —
  this is a data-entry error, not an invalid transaction, so it's fixed
  instead of discarded.
- Dates are parsed with `pd.to_datetime`; unparseable dates are dropped.
- Exact duplicate rows are removed.

Every drop or fix is logged with a count, so nothing is silently changed.

**Analysis (`analyze_data()`):**
- Revenue by product and by region via `groupby(...)["Total_Sales"].sum()`, sorted descending.
- Monthly revenue via a `YYYY-MM` period column, sorted chronologically.
- Month-over-month % change between the two most recent months.
- Average order value, unique customer count, and revenue per customer.
- Best-selling product by *units* (separate from best by *revenue* — a
  cheap product can outsell an expensive one in volume without leading in revenue).

**Visualization (`matplotlib`):**
- **Bar chart** — total revenue per product, with value labels on each bar.
- **Pie chart** — percentage share of revenue by region, using `autopct`.
- **Line chart** — monthly revenue over time, with point markers and value
  annotations, to show the trend clearly.

All charts use consistent titles, axis labels, and gridlines for readability,
and are saved at 150 DPI so they're crisp when embedded in the report.

## 6. Testing Evidence

**Happy path** — `python main.py` on the full `sales_raw.csv`:
- 633 raw rows → 624 clean rows after removing 3 missing-field rows, 3
  non-numeric rows, 2 invalid-quantity/price rows, and 1 duplicate row, plus
  recalculating 1 row with a mismatched `Total_Sales` (verified against the
  intentionally-seeded dirty rows in `generate_data.py`).
- All 3 chart files and the report were generated with no errors.

**Error handling** — verified directly against the pipeline functions:

```python
>>> load_data("data/does_not_exist.csv")
FileNotFoundError: Could not find data file at 'data/does_not_exist.csv'. ...

>>> clean_data(pd.DataFrame({"Date": ["x"], "Product": [""], "Quantity": [""],
...                          "Price": [""], "Customer_ID": ["C1"], "Region": [""],
...                          "Total_Sales": [""]}))
ValueError: No valid rows remained after cleaning. Check the raw data file.
```

Both cases are caught in `main()` and produce a clean error message and exit
code 1, instead of a raw traceback.

**What's covered:**
- Missing data file
- Empty data file
- Missing required columns
- Missing Product/Region values
- Non-numeric Quantity/Price/Total_Sales
- Negative/zero Quantity or Price
- Unknown region names
- `Total_Sales` not matching `Quantity x Price`
- Unparseable dates
- Duplicate rows
- All-invalid dataset (zero rows survive cleaning)

## Sample Insight (from the generated report)

Laptop is the top revenue-generating product despite likely not being the
top seller by units — a reminder that revenue leadership and volume
leadership are two different things, and pricing strategy matters as much
as sales volume. See `report/analysis_report.md` for the full breakdown,
including the region and monthly trend numbers.
